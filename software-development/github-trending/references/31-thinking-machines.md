# Thinking Machines — Expert Judgment Replication

**Source:** https://thinkingmachines.ai/news/learning-to-replicate-expert-judgment-in-financial-tasks/
**Company:** Thinking Machines (Mira Murati)
**Category:** ML Research / Expert Distillation

## Summary

Thinking Machines превратила закрытую экспертную оценку Bridgewater (хедж-фонд Рэя Далио) в обучаемый навык и обошла frontier-модели, снизив число ошибок на 29.8%.

### Задача

ИИ подавали финансовые статьи, отчёты, документы центробанков, письма. Модель должна была определить, что аналитик должен прочитать первым. Для LLM это оказалось тяжёлой задачей — при обычных промптах модели держались на уровне 46–50% accuracy (почти как угадывание).

### Что сработало

- **Экспертные промпты** поднимали качество до 74–78%
- **Разметка от опытных инвесторов Bridgewater** дала лучший результат
- Неэкспертные метки провалились: важны рыночный контекст, приоритеты и профессиональное чутьё
- Один заголовок про тарифы может быть сильным сигналом, другой громкий геополитический инфоповод останется шумом

### Очистка данных

Bridgewater чистила датасет через спорные кейсы. Если модель расходилась с разметкой — пример возвращали экспертам на повторную проверку. Так в обучение попали паттерны принятия решений, которые эксперты видят интуитивно, но редко могут описать полноценной инструкцией.

### Три приёма обучения

1. **Смешивание типов задач** — модель училась применять экспертную оценку в похожих сценариях, а не запоминать один узкий шаблон
2. **Ограничение обновлений при обучении** — снижение риска, что модель зацепится за случайные признаки и начнёт уверенно ошибаться
3. **Self-distillation** — модель дообучали на ответах более сильных версий; лучшие checkpoints постепенно становились учителями для следующих итераций

### Результат

- **На 29.8% меньше ошибок**, чем у лучшей frontier-модели
- **В 13.8 раза ниже inference cost**
- Конкурент не скачает такой датасет с Hugging Face — качество экспертных решений — конкурентное преимущество

## What to Take

| Принцип | Применение |
|---------|-----------|
| Экспертная разметка > промпт-инжиниринг | Для сложных доменов качественные метки важнее красивых промптов |
| Очистка через спорные кейсы | Если модель расходится с разметкой — пересматривать разметку, не слепо верить модели |
| Смешивание типов задач | Обучение на разнообразных сценариях → обобщение, не запоминание шаблона |
| Ограничение обновлений (conservative updates) | Регуляризация при fine-tuning — не давать модели переобучаться на шум |
| Self-distillation (итеративное улучшение) | Лучшие checkpoints как учителя для следующих итераций |
| Датасет как moat | Качественные экспертные данные — конкурентное преимущество, которое нельзя скопировать |

## Applicability to Projects

- **DBDPerksAddonReveal** — паттерн "очистка через спорные кейсы": если CLIP ошибается на конкретном аддоне, пересматривать разметку, не просто добавлять больше данных
- **MPT** — если есть экспертные правила приоритизации входящих данных, их можно дистиллировать в модель через разметку, а не через промпты
- **Hermes Agent** — self-distillation: лучшие результаты cron-задач как референсы для следующего запуска
- **Универсально** — для любой задачи, где LLM плохо справляется с промптами: попробовать экспертную разметку вместо промпт-инжиниринга
