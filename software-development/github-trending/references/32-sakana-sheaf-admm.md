## Summary

Sakana AI представила метод координации_multi2агентов через Sheaf-ADMM — объединение ADMM (distributed optimization) и sheaves (applied topology). Интеллект рассматривается как распределённая система: сложная задача делится на пересекающиеся фрагменты, каждый агент работает только со своей частью, затем агенты согласуют решения на границах.

### Как работает

1. **Локальное решение** — агент предлагает решение для своей части задачи
2. **Сверка с соседями** — на границах (пересечениях областей) агенты сглаживают конфликтующие части
3. **Память конфликтов** — если согласия нет, конфликт сохраняется и влияет на следующий раунд переговоров
4. **Итеративная сходимость** — координация в несколько раундов до общего результата

Ключевое отличие: агенты не обмениваются всей информацией — только согласуют на пересечениях.

### Результаты

| Задача | Sheaf-ADMM | Baseline | Разница |
|--------|-----------|---------|---------|
| Multi-Agent Sudoku (видят только строку/столбец/блок) | **93%** | 11% (message-passing) | +82% |
| Image classification, domain shift (MNIST canvas size) | **86%** | 11% (CNN) | +75% |
| Maze pathfinding | = baseline | = | 5D канал vs 42D (8.4× меньше) |

### Ключевые свойства

- **Прозрачность** — видно как агенты спорят, корректируют решения, приходят к результату (не hidden states)
- **Эффективность коммуникации** — 5-мерный канал вместо 42-мерного при той же точности
- **Robustness к domain shift** — сохраняет 86% accuracy при смене размера canvas, где обычная CNN падает до 11%
- **Доказанная сходимость** — ADMM имеет теоретические гарантии из convex optimization

### Теоретическая база

- **ADMM** (Alternating Direction Method of Multipliers) — классический метод distributed optimization, разделяет сложную оптимизацию на локальные подзадачи + согласование
- **Sheaves** (пучки) из applied topology — математическая структура для описания данных на пересекающихся областях, локальные данные + правила склейки на границах

## What to Take

| Принцип | Применение |
|---------|-----------|
| Разделяй и согласовывай на границах | Сложную задачу бьём на части, согласуем только на пересечениях — не передаём всё |
| Память конфликтов | Неразрешённые конфликты влияют на следующий раунд — не забываем, а используем |
| Прозрачная координация | Видеть процесс согласования, а не чёрный ящик hidden states |
| Малоразмерный канал коммуникации | 5D вместо 42D при той же точности — компрессия смысла, не данных |
| Sheaf theory как каркас | Локальные модели + правила склейки = глобально согласованный результат |

## Applicability to Projects

- **DBDPerksAddonReveal** — двухэтапное распознавание (killer → addons) можно рассматривать как multi-agent: один агент идентифицирует killer, другой — addons из его пула, согласование на границе (killer-addon соответствие)
- **PIVOBOT** — если несколько ботов/сервисов обрабатывают разные части запроса, Sheaf-ADMM подход для согласования без обмена всей информацией
- **MPT** — распределённая обработка: разные микросервисы работают со своими данными, согласование на пересечениях (user sessions, shared state)
- **Hermes Agent** — multi-agent delegation: subagents работают изолированно, согласование только на границах (общие файлы, общие данные) вместо полного обмена контекстом
- **Vetka_dwg** — CV→CAD: разные этапы (детекция линий, распознавание текста, классификация слоёв) как агенты, согласование на пересечениях элементов чертежа
